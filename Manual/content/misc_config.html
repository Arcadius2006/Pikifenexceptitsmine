This article explains how miscellaneous configuration files on Pikifen work. These are plain engine data files.

Game configuration

This file controls some general settings about how the game works. It can be found in Game_data/Misc/Config.txt. Create it if it does not exist.

The following properties exist:

Basics

game_name Name of the fangame. Text
game_version Version of the fangame. Can be anything, but the preferred format is something like "1.2.3". Text

Carrying

carrying_color_move Color to use for the carrying numbers when an object that is meant for the ship is being carried. Color 255 255 255
carrying_color_stop Same as above, but for when the object is not moving. Color 102 191 191
carrying_speed_base_mult When an object is being carried, its movement speed depends on the carriers' average speed (and more). With 0 carriers, the average speed is multiplied by base_mult, and with all spots taken, it's multiplied by max_mult. This property controls the base value. In other words, this controls how high the slowest speed of a carried object is. Number 0.5
carrying_speed_max_mult Same as above, but the max_mult. In other words, this controls how high the fastest speed of a carried object is. Number 0.8
carrying_speed_weight_mult For every 1 unit of weight, decrease the carrying speed by this much. Number 0.0004

Day time

day_minutes_start Time of day when gameplay starts. This follows a format similar to real life, so 0 is midnight, 60 is 1 AM, 120 is 2 AM, 420 is 7 AM. This is only used for the Sun Meter HUD item. Number 420
day_minutes_end Same as above, but for when gameplay ends. Number 1140

Pikmin basics

pikmin_order Preferred order to sort Pikmin with. Used when swapping Pikmin, dismissing them, organizing them in an Onion, etc. Types you do not specify will be sorted alphabetically and go to the end of the list. Text list
standard_pikmin_height The height of a "standard" Pikmin, i.e. a Pikmin that's not shorter or taller than most. Used to help calculate some physics logic. Number 24
standard_pikmin_radius The radius of a "standard" Pikmin, i.e. a Pikmin that's not skinnier or fatter than most. Used to help calculate some physics logic. Number 5

Leader basics

leader_order Like pikmin_order, but for the leaders. Text list
standard_leader_height Like standard_pikmin_height, but for the leaders. Number 46
standard_leader_radius Like standard_pikmin_radius, but for the leaders. Number 16
spray_order Like pikmin_order, but for sprays. Text list

Pikmin details

idle_bump_delay In order for an idle Pikmin to be bumped, it must be away from the active leader for these many seconds first. Number 5
idle_bump_range Minimum distance between an idle Pikmin and the active leader for the Pikmin to be called to the group. Number 50
idle_task_range Minimum distance between a Pikmin and a task before the Pikmin can take the task, when the Pikmin is idling. Number 50
swarm_task_range Same as above, but for a Pikmin in the group during swarming. Number 3
max_pikmin_in_field Maximum number of Pikmin that can be on the field. Number 100
maturity_power_mult Every maturity level, add this much to a Pikmin's attack power, as a ratio (e.g. 0.1 adds 10% more). Number 0.1
maturity_speed_mult Same as maturity_power_mult, but for the movement speed. Number 0.1
pikmin_chase_range Pikmin will chase after enemies unless they're this far away. Number 200

Leader details

can_throw_leaders Whether it's possible for leaders to throw other leaders. Boolean true
cursor_max_dist Maximum distance away from the leader that their cursor can go to. Number 200
cursor_spin_speed How quickly the mouse cursor's icon spins, in degrees per second. Number 180
next_pluck_range When auto-plucking, a leader will only choose the next Pikmin if it is within this distance. Also affects the range at which idle leaders will go pluck. Number 160
no_pikmin_color Color that represents the absence of Pikmin. Color 191 230 230
onion_open_range Leaders can open an Onion if they're within this range from the center of the Onion object. Number 24
group_member_grab_range A leader can grab the closest group member only if it is within this distance. Number 64
pluck_range A leader can pluck a Pikmin only if it is within this distance. Number 30
throw_max_dist Maximum distance away from the leader that a throw can be aimed to. Number 200
whistle_growth_speed How quickly the whistle's radius grows in size, in pixels per second. Number 180
whistle_max_dist Maximum distance away from the leader that the whistle can focus on. Number

Misc.

message_char_interval Time interval between characters when displaying a message, in seconds. 0 shows the entire text instantly. Number 0.03
zoom_max_level Maximum level of zoom (i.e. how close the camera can get). Number 3
zoom_min_level Minimum level of zoom (i.e. how far away the camera can get). Number 0.66

Spike damage

"Spike damage" is a mechanic where objects that suffered damage can immediately retaliate by damaging their attacker. In the Pikmin games, this is used by a White Pikmin to release poison to the enemy that ate it, in order to cause poison damage. In the engine, the same can be done, but any object can retaliate in any way, whether it is swallowed or damaged normally.

The folder Game_data/Spike_damage_types controls the types of spike damage that exist in the game. Each data file represents one type of spike damage, and the properties inside control how it works. The name of the file does not matter.

The properties are:

Common

  • All of the common content properties.

Others

damage How much damage to cause. What exactly this number means depends on the is_damage_ratio property. Number 0
ingestion_only true if the spike damage should only be caused when the object is ingested, false if it's meant to be caused when the object is hurt normally. Ingestion only works for Pikmin. Boolean false
is_damage_ratio If true, then the number specified in damage is the ratio of total health to sap away. For instance, 0.5 means one half of the enemy's total health is removed. If false, the number is just a normal amount of damage. Boolean false
particle_generator When the damage is applied, some particles can be spawned. This is the name of the particle generator to use, if any. The particles are spawned in the position of the object that receives the damage. Text
particle_offset If particles are meant to spawn, you can offset them from the object's center here. The format is x y z, and these are relative coordinates. Text
status_to_apply A status effect to apply when the spike damage is dealt. Leave empty for none. Text

Remember to change the object type properties for the objects you want to deal this spike damage, as well as the different spike damage vulnerability modifiers, as explained here.

System asset file names

Some graphics and sounds are used by the engine in several ways, instead of used specifically for a user-made enemy, area, etc. Inside the file Game_data/Misc/System_asset_file_names.txt you can specify the file name (and path) of some or all of these assets. This data file has two blocks, graphics and sounds, and each block has a list of properties, where each property's name is the name of the asset, and the value is the file name. The files must be inside of the folder Game_data/Graphics and Game_data/Audio/Sounds. Example: mouse_cursor = Asterisk.png will make the mouse cursor in-game use the Asterisk.png image.

Fonts

These properties belong to the fonts block:

area_name Font for the area's name in loading screens. Text Area_name_font.png
counter Font for HUD counters. This only needs a dash, a dot, a slash, numbers, a colon, and a multiplication sign. Text Counter_font.png
counter Font for the counter displayed next to the cursor. This only needs numbers. Text Cursor_counter_font.png
slim_font Font for slim text. Text Slim_font.otf
standard Font for standard text. Text Font.png
value Font for the carrying / money values. This only needs a currency icon, a dash, and numbers. Text Value_font.png

Audio

camera Camera zoom level change click. Text Camera.ogg
menu_activate Sound of a GUI item in a menu being activated. Text Menu_activate.ogg

menu_select Sound of a GUI item in a menu being selected. Text Menu_select.ogg
pluck Ground pop for when a Pikmin gets plucked. Text Pluck.ogg
spray A leader using a spray. Text Spray.ogg
switch_pikmin Plop for when the current standby Pikmin type is switched. Text Switch_Pikmin.ogg
