The following is a general changelog for Pikifen. This changelog will only list the most notable differences between each version of the engine; for a more detailed list of changes throughout the development period, check the commit log on GitHub.

Some versions may be unable to read content created in older versions. If you want to keep your content from one version to the next, read the "Content making changes" list in that update's section.

Versions are numbered after the format major.minor.revision. The major number increases when the project is radically different from the major version before it. The minor number increases every time there are a bunch of changes. The revision number increases when there are bug fixes on the revision before it, and does not break compatibility.

0.23.0

Unreleased – the "Mission Mode" update

Gameplay changes

  • Missions:
    • Added Mission-type areas. Gameplay during missions usually involves reaching a goal, avoiding some fail conditions, and getting medals based on several scoring criteria.
    • Added Info about the area that is currently selected in the list, in the area selection screen.
    • Added Mission information tab in the pause menu, if you're playing on a mission.
    • Added A "play" button and a "make" button for the main menu, which show the list of area types that can be played, and the list of editors, respectively.
    • Changed The results screen was completely revamped to better suit a mission's completion screen.
  • Others:
    • Added The ability to play when there are no available leaders, or even if there were none to begin with. There isn't much to do in this state other than move the camera, however.
    • Added The ability for the camera to be influenced by the group and the cursor. Added an option to control how much the cursor influences it.
    • Added Lifetime statistics menu.
    • Added A confirmation message for the pause menu, shown when you want to leave. This can be changed in the options. (Thanks Neo)
    • Changed Misc. pause menu improvements.

Content making changes

  • Area editor improvements:
    • Added History of opened folders.
    • Added "Gameplay" panel, with the starting spray amounts (with a better interface), as well as the new mission stuff.
    • Added Description, tags, thumbnail, and difficulty properties for areas.
    • Added Show path link length option. (Thanks Nataly)
    • Added Show circular sector info option. (Thanks Neo)
    • Added Basic/advanced interface mode option.
    • Added The ability to choose what type and label new path links will have as you're drawing them. (Thanks Nataly)
    • Added Highlights to a sector, mob, etc. when you hover your mouse over it. (Thanks Helodity)
    • Added Toolbar buttons to swap to the layout, objects, paths, or details mode. (Thanks Nataly)
    • Added H shortcut key to change a sector's height.
    • Added T shortcut key to copy a sector's texture.
    • Added L, O, P, D shortcut keys to swap to the layout, objects, paths, and details mode. Advanced interface mode only. (Thanks Nataly)
    • Changed Areas can now specify at what point of the in-game day they start, as well as day time speed or (for missions with time limit) the time of day they end. The day_minutes_per_irl_sec field was removed from the config file since it is no longer needed.
  • Maker tools:
    • Added Ctrl+F12 now changes the auto-start maker tool's data to match the current mode and currently open content.
    • Added "Show collision" maker tool. (Thanks Helodity)
  • New properties:
    • Added A points property to enemy types.
    • Added A states_ignoring_hazard property for scripts.
    • Added A hold_offset_vertical_distance property for a mob type's children data.
    • Added A custom_category_name property for mob types. With this, the pre-packaged mob types are now better organized in the editors.
    • Added A can_walk_on_others property for mob types.
    • Added knocked_down_duration and knocked_down_whistle_bonus properties for Pikmin types.
    • Added An auto_shrink_smallest_radius property for pile types.
    • Added An interval_deviation property for particle generators.
  • Others:
    • Added GUI editor. This should make it easier to position and resize GUI elements in menus and HUDs.
    • Added The ability to lock the center (as opposed to the opposite side/corner) by holding Alt, when you are dragging a side/corner handle of a transformation widget in an editor's canvas.
    • Changed In editors, keyboard shortcuts made for the canvas will now work anywhere, so long as you aren't writing in a text box. (Thanks Neo)
    • Changed Drag buttons in editors now only need a single click in order to enter text input mode.

Custom content compatibility warnings

  • Revamped the following GUIs:
    • Changed The main menu's GUI got a complete revamp.
    • Changed The area menu's GUI got a complete revamp.
    • Changed The results screen's GUI got a complete revamp.
    • Changed The pause menu's GUI got a complete revamp.
  • Others:
    • Changed Old area folders used to be in Game_data/Areas/<folder>, but must now be moved to Game_data/Areas/Simple/<folder>. If you care about auto-backups and reference image data, do the same to the area-related stuff in the User_data folder.
    • Removed The system asset for the Pikmin silhouette has been removed in favor of two leader silhouettes.
    • Added Two leader silhouette graphics to replace the Pikmin silhouette image.
    • Added standard_leader_height and standard_leader_radius game config properties.

0.22.0

July 5th, 2022 – the "polish" update

Gameplay changes

  • Added a help page to the pause menu, with gameplay information, tips, controls, Pikmin type explanations, and more.
  • White Pikmin poison now applies a status effect that drains health over time. Enemies still end up losing the same amount of health in the end.
  • Pikmin are now better at detecting gates and other rectangular objects.
  • Pikmin can now circle around enemies when in combat.
  • Bridges now grow in chunks, and the point Pikmin deliver fragments to, or the position of the pile of sticks, now moves along with the bridge.
  • Misc. graphical enhancements:
    • All gates have received graphical improvements.
    • Decorative objects no longer cast light in blackout weather.
    • When Pikmin deliver an object to an Onion, ship, etc., the sucking animation is now more dynamic. Also, added a tossing animation for delivering bridge fragments to bridges.
    • Improved the look of the ship's beam and the leader's whistle. The pretty_whistle option has been removed. (Thanks Helodity)
    • HUD elements now animate when things change, and slide off-frame when menus are entered in-game. (Thanks Helodity)
    • In-game message boxes are now much more animated.
    • Health wheels and fraction numbers now appear and disappear smoothly, and are more animated in general.
    • Enemies now squash and stretch when damaged.
    • Objects being carried now sway.
    • Icons for controls have been improved.
  • Fixed leaders being able to grab group members from the other side of walls or objects with collision, like gates.

Content making changes

  • Bridges are now made up of objects. This should make the process of placing bridges in an area much simpler, since there's no need to fiddle around with sectors. Areas with bridges need to be updated to use this new bridge format.
  • Status effects now have a health_change property, which controls how much health the object loses per second in an absolute value, instead of it being a ratio of total health.
  • Spike damage types now have a status_to_apply property, which allows the damaged object to receive a status effect.
  • Object types now have a status_vulnerabilities block, which works like the other vulnerability blocks, but controls damage taken from status effects.
  • Object types now have a terrain_radius property, which controls their collision against walls.
  • Object types now have a description property, which can be used to briefly describe it. This appears in the area editor, and Pikmin types make use of this for the help page of the in-game pause menu.
  • The tooltip of an object type's area editor property now word-wraps automatically. If you relied on line breaks (\n) to do this before, you may want to adjust it.
  • The set_animation script action now accepts the options random_time and random_time_on_spawn, to let the animation start on a random time, or start on a random time but only if the mob just spawned.
  • Added the on_receive_delivery object script event.
  • Added the health_ratio option to the get_info object script action.
  • Added the health script variable.
  • Added the max_health script variable.
  • Added the bright_circle and bright_ring system assets.
  • The area editor now also shows the terrain radius of selected objects (if the "Show territory/terrain radius" option is on).
  • Converters now use an animation called bumped instead of bumping, for consistency's sake.
  • System assets no longer have icons for mouse-related controls, but instead have one spreadsheet for many special control icons.
  • The gameplay HUD received all sorts of new items, and some have been renamed to be more consistent.
  • Areas and animations now save the engine version they were edited in. This will likely be used in the future to allow the engine to automatically port older content to newer versions.

0.21.0

October 22nd, 2021 – the "Pikmin 3" update

Gameplay changes

  • Added Rock Pikmin.
    • When thrown at an enemy, they cause damage when they collide, and then bounce back.
    • They also lunge forward on their own when they get close to an enemy they are chasing on foot. This impact works just like a throw, bounce back included.
    • Also, updated the pellets, Pellet Posies, Candypop Buds, and Onions to support Rock Pikmin.
    • Plus, added cystal walls, crystal nodules, large crystals, small crystals, and Skutterchucks.
  • Added Winged Pikmin.
    • If their target is above or below them, they can move up and down in the air to get to it.
    • When following a leader, they try to stay a bit above the leader, and when idling, they also try to idle some distance above-ground.
    • Because they are flying in the air when following a leader, they can travel over bodies of water. That said, leaders that are underwater can't grab or throw them.
    • Also, updated the pellets, Pellet Posies, Candypop Buds, and Onions to support Winged Pikmin.
    • Plus, added bamboo gates, Flukeweeds, and Puffy Blowhogs.
  • Added Bulbmin, both adults and young. They are invulnerable to all elemental hazards, but the young lack an Onion or ship to go to.
  • Added Alph, Brittany, and Charlie, as well as the S.S. Drake. The Hocotatian leaders also got a glow-up. The President's color was changed to yellow, otherwise half of the leaders would be blue.
  • Purple Pikmin now cause a bit of collision damage when they land on an enemy after being thrown.
  • Enemies petrified with the ultra-bitter spray now shake before breaking free, their petrification visually bursts when it's over, and their animations now properly freeze.
  • Munge Dweevils can now pick up treasure.
  • Pikmin will now only take paths that they can survive. e.g. a carcass with 1 Red Pikmin and 9 Blue Pikmin carrying it will take a path that doesn't go through water, but a carcass carried by 10 Blue Pikmin might take that water path.
  • Objects now move with acceleration. This should make them a bit smoother when moving.
  • Revamped the cursor.
    • The leader cursor's color now matches the standby Pikmin's color. If there is none, it uses a faded cyan color.
    • The number of Pikmin of the standby type now also appears next to the cursor.
    • There is now a line connecting the leader to the cursor. When the leader has a group member, the line changes to represent a preview of how that member would be thrown. This helps to show which ledges can and can't be reached.
    • Added an option to set the cursor's speed, used when controlling it without a mouse. (Thanks MonkeyUhOh)
  • If a Pikmin is lying down, a leader can now whistle it to make it get up faster, and return to the leader once it does.
  • Added an option to allow the leader to auto-throw. If you prefer not to mash the throw button, consider turning this on.
  • If a new leader shows up from a defeated enemy, their HUD information will now display properly. (Thanks pikminjosh)
  • All menus now have a tooltip for the currently selected item.
  • The in-game message box now looks like other boxes, instead of a stretched flat image.
  • Fullscreen mode now uses a simple borderless window. True fullscreen mode can still be enabled in the options file, but comes with risks.

Content making changes

  • Rock Pikmin-related features:
    • Added a property for Pikmin types, attack_method, that controls how the Pikmin attacks. If set to impact, that Pikmin attacks like a Rock Pikmin, as explained above.
    • Their on-foot lunges use the attack animation, so it should ideally be in a pose where they lunge forward, and have a hitbox.
  • Winged Pikmin-related features:
    • Added a property for Pikmin types, can_fly, that makes a Pikmin fly like a Winged Pikmin, as explained above.
    • These Pikmin can only move vertically if the state they are in allows it (e.g. they can't fly if they're being knocked back).
    • Technically, if a leader is on a floor with a hazard (liquid or otherwise), they will be unable to grab Pikmin that are outside it and are vulnerable to the hazard. If they grab and bring those Pikmin to said floor, the Pikmin will suffer the effects.
  • Pikmin now have a new state that is entered when they land on a mob, mob_landing. In it, they change to an animation of the same name. It is only after this animation ends that they latch on or bounce back. As such, this animation is a good opportunity to get the Pikmin to cause damage upon collision.
  • Revamped paths.
    • If a path stop is on top of a sector with hazards, objects will now be unable to traverse it if they are vulnerable to the hazards. In the case of a carried object, this applies to the carrier Pikmin. If there is another path, they will take it, otherwise they will stay stuck in front of that path stop. If the hazard is only on the floor, and the object or carrier Pikmin can fly, then they will still take that path.
    • Path links now have a type. "Normal" works like normal, "custom only" can only be used by objects that need that path for scripting, "light load only" can only be taken by objects carrying a weight of 1 or nothing, and "airborne only" can only be taken by flying objects.
  • Revamped the status system.
    • Added the animation_change property, which lets the object's animation be changed when the status is applied.
    • Added the freezes_animation property. If true, the object will stay stuck in its current sprite until the status effect is removed.
    • Added the reapply_rule property, which controls what happens to the auto-remove time of a status effect in case it is re-applied.
    • Added the remove_on_hazard_leave property. This is now used for statuses like drowning, where the status effect is removed once the object leaves the hazard that was applying the status effect.
    • Added the replacement_on_timeout property, which lets a status effect be swapped for another when its time runs out.
    • Added the state_change_type and state_change_name properties, so status effects can change the object's state when applied.
    • Added the shaking_effect property, which makes the object shake left and right while under the effects of the status effect.
    • Object types, in their hazard vulnerability data and spike damage vulnerability data, can now be told to receive a status effect when interacting with one.
    • Plus other changes noted below in the compatibility warnings.
  • New mob script actions:
    • load_focused_mob_memory and save_focused_mob_memory let the currently focused object be saved into a memory, and be recalled later. This allows the object to keep track of more than one object at a time, although it still only focuses on one at a time.
    • link_with_focused_mob lets the object be linked to whatever object it has in focus.
    • hold_focused_mob lets the object hold whatever object it has in focus, much like how a leader holds a Pikmin.
    • throw_focused_mob lets the focused object be thrown in an arc towards a destination.
    • store_focus_inside lets the object store another one inside, to be released later.
    • release_stored_mobs lets the object release any other object that is stored inside.
    • get_angle lets the script obtain the angle between two sets of coordinates.
    • get_coordinates_from_angle lets the script obtain the X and Y coordinates calculated from a given angle and distance.
    • get_distance lets the script obtain the distance between two sets of coordinates.
    • get_floor_z lets the script obtain the Z coordinate of the floor at the specified coordinates.
    • set_can_block_paths lets the object block or stop blocking the paths of others.
    • set_flying lets the object move in mid air at will, unaffected by gravity.
    • set_radius lets the object's radius be changed.
    • set_shadow_visibility lets the object's shadow be hidden.
    • follow_path_to_absolute lets the object follow path stops to get to the specified coordinates.
    • follow_path_randomly lets the object follow path stops to get to a random stop in the area.
  • Other mob script changes:
    • The move_to_absolute and move_to_relative actions can now use an optional Z coordinate argument.
    • The turn_to_absolute and turn_to_relative actions can now either use an angle argument, or coordinate arguments.
    • Added group_task_power and angle to the get_info action, so a group task object can obtain the current "Pikmin power" applied to it, and so an object can obtain its current angle, respectively.
    • The on_hitbox_touch_a_n event used to trigger regardless of the first hitbox's type. This has now been fixed, and on_hitbox_touch_n_n was created for when the both hitboxes are of the normal type.
  • Group task object type improvements:
    • Added the flying_pikmin_only property.
    • Added the spots_z property to allow control of the vertical position of the contribution spots.
    • Added the carrying value to the worker_pikmin_pose property (though it doesn't change anything yet).
  • New object type properties:
    • acceleration controls the object type's acceleration.
    • has_group lets the object have a group of other objects (usually Pikmin) following it.
  • New game configuration properties:
    • no_pikmin_color controls the color the cursor shows when no Pikmin are available.
    • throw_max_dist and whistle_max_dist control the maximum distances at which the leader can throw or whistle.
  • Minor editor tweaks.
    • In the animation editor's sprite or animation picker, the buttons now shows thumbnails of the sprite or the animation's first frame.
    • Double-clicking in the area editor while in object editing mode now creates a new object where you clicked.
    • In the area editor, you can now specify a different horizontal and vertical scale, when resizing everything.
    • Path links in the area editor are now colored based on their type, and whether they are one-way.
    • Revamped the area editor's problem finder and path preview.
    • You can now delete an area inside the area editor.
  • Added the group_pikmin_only and affects_user properties to spray types. One controls whether a spray affects only the Pikmin in the group or leaders as well, and the other controls whether the leader that used the spray gets sprayed too.

Custom content compatibility warnings

  • Animation changes. These are needed in order for Pikmin to get up when whistled, and for Purple Pikmin and Rock Pikmin to cause collision damage:
    • lying is now just the Pikmin lying down forever.
    • knocked_back is a new animation for the Pikmin getting knocked back forever.
    • getting_up is a new animation for the Pikmin getting up from lying down.
    • mob_landing is a new animation for the thrown Pikmin landing on a mob and getting ready, and/or causing damage.
  • Status effect changes:
    • Turned the disabled_state_inedible status effect type property to turns_inedible instead. This way, any status can make a Pikmin inedible while under its effects.
    • The causes_* status effect type properties were removed in favor of the state_change_type property. Anything related to the Pikmin's "disabled" state has now also been renamed to the more apt term "helpless".
    • The on_hitbox_touch_a_n event was corrected, so if you relied on the incorrect behavior, use on_hitbox_touch_n_n instead.
    • The animation and animation_mob_scale properties have been renamed to overlay_animation and overlay_anim_mob_scale, respectively.
  • Converters now need hitboxes for the Pikmin to touch when thrown.
  • The blocks_carrier_pikmin object type property has been renamed to can_block_paths.

0.20.0

April 24th, 2021 – the "missing menus" update

Changes

  • Revamped Onions.
    • Interacting with an Onion now displays a menu to manage the Pikmin with.
    • Pikmin now have logic to climb up an Onion's legs and enter, or to come out and slide down a leg.
    • An Onion can now manage multiple Pikmin types, not just one.
    • Ships can now store and manage Pikmin, just like Onions.
    • The pre-packaged Onion graphics now animate, and look like the more modern Onion designs.
    • Added a new Onion type that manages all three – the Master Onion.
  • Added a results screen.
    • When the player finishes playing in an area, the results screen gives them some basic stats: time taken, treasure points obtained, treasure points available, enemies killed, enemies available, Pikmin born, and Pikmin deaths.
    • The player is also given the option to retry, keep playing after hours, or pick a different area.
    • The results screen is entered when the player chooses to end the day via the pause menu, when the day time is over, or when all leaders are KO.
    • Added a new property for treasure types: points. Use this to specify how many points they are worth.
  • Added a pause menu. Players can retry, end the day, or quit to the area selection menu from here.
  • Revamped GUI.
    • Updated all menus to have a better and more unified look.
    • The position of GUI items in all menus can now be customized, just like the HUD was customizable until now.
  • Revamped edge aesthetics.
    • You can go to the area editor, select edges, and edit their properties.
    • With this, you can control how and if walls cast a shadow, as well as their color and length. (Thanks Scruffy, Neo)
    • You can also now make the higher sector of a wall look smooth at the ledge, instead of the ledge abruptly cutting off to the lower sector. (Thanks Tenacious)
    • Finally, bodies of water will now have a simple foam effect around their limits.
    • Wall shadows have been revamped to work better in general.
  • Expanded the get_info action:
    • Added new data: x, y, and z. Use these to get the object's coordinates. (Thanks Kman)
    • Also added focus_distance. Use this to get the distance between this object and its focused object.
    • Added a new action: get_focus_info. It works like get_info, but when applicable, gets info from the focused mob instead. (Thanks Kman)
  • Added a mob script event, on_leave_hazard, and added a way to get the name of the hazard via the get_info action, both for this event and on_touch_hazard. (Thanks Kman)
  • Pikmin now push each other slightly.
  • Health wheels now transition smoothly to their target value, and also fade away when they are empty. (Thanks Kman)
  • Added new system assets: bubble_box, used to draw box-like bubbles in the HUD and menus, focus_box, used to show which GUI item is focused, and more, used for menus to indicate there's more content beyond.
  • Holding Shift when the snapping mode is "nothing" will now snap to the grid in the area editor. The snapping mode is now also saved to the options. Finally, Shift+Plus and Shift+Minus change the grid spacing. (Thanks Nutty171, Neo)
  • Players can now customize the colors of the GUI widgets in the editors. (Thanks Kman)
  • The Beady Long Legs's logic was slightly cleaned up.
  • Revamped code and added instructions to make it possible to compile on Mac OS. (Thanks MidnightWhisper)

Compatibility warnings

  • Onion types now use the pikmin_types property instead of pikmin_type.
  • The mouse cursor graphic now displays at full-size, all the time. If you made a custom one, you'll probably want to resize its image file to 70% its old size.
  • The pre-packaged Pellet Posy graphics files are now split per-type, as opposed to per-size. This makes more sense for adding new types of Pikmin/pellet. (Thanks Kman)
  • The counter font now also includes a dot and a colon. If you changed this font, please update it accordingly.
  • The gameplay HUD information is now stored in Game_data/GUI/Gameplay.txt, and the title screen logo information now goes in Game_data/GUI/Main_menu.txt.
  • The sector property "always casts shadow" has been removed, and wall shadows should be slightly less dark than before in general. If you have walls that you need to look the same as before, you'll want to go to the edge properties and tweak their shadow's length and color.
